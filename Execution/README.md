# Experiment 5 — Single Cycle MIPS Processor Program Execution (Verilog HDL)

## Overview

This experiment focuses on implementing and demonstrating the execution of a **single-cycle 16-bit MIPS processor** using Verilog HDL. All previously designed components — ALU, Control Unit, Register File, Instruction Memory, and Data Memory — are integrated to execute an encoded assembly program.

The assembly instructions are manually encoded and stored in the instruction memory. The processor fetches, decodes, and executes instructions sequentially in a single clock cycle per instruction.

The execution is verified using simulation in Xilinx Vivado and demonstrates processor functionality for R-type, Immediate-type, and Branch-type instructions.

---

## Aim

To encode an assembly program in instruction memory containing at least one R-type, one Immediate-type, and one Branch-type instruction, and demonstrate its execution using a single-cycle MIPS processor in Verilog HDL.

---

## Objectives

- Integrate all datapath and control components into a single-cycle MIPS processor  
- Encode assembly instructions into machine code  
- Store instructions inside instruction memory  
- Execute instructions sequentially using processor control logic  
- Implement branching and program flow control  
- Verify execution using a Verilog testbench and simulation  
- Demonstrate processor behavior using Vivado simulation and FPGA implementation  

---

## Processor Architecture

The processor follows a **single-cycle architecture**, where each instruction completes execution within one clock cycle.

### Integrated Components

- Instruction Memory
- Control Unit
- Register File
- ALU Control Unit
- Arithmetic Logic Unit (ALU)
- Data Memory
- Jump and Branch Control Logic
- Program Counter (PC)

All modules operate together to perform instruction fetch, decode, execute, memory access, and write-back stages.

---

## Modules Implemented

### 1. Instruction Memory

Stores encoded machine instructions representing an assembly program.

The program includes:
- Load instruction
- Immediate comparison
- Branch instruction
- Arithmetic operations
- Loop execution using branch logic

The processor fetches instructions using the Program Counter (PC).

---

### 2. Control Unit

Generates control signals based on instruction opcode.

**Control Signals Generated**
- `reg_dst`
- `mem_to_reg`
- `alu_op`
- `jump`
- `branch`
- `mem_read`
- `mem_write`
- `alu_src`
- `reg_write`
- `sign_or_zero`

These signals control datapath execution for different instruction formats.

---

### 3. Register File

- Contains 8 general-purpose registers
- Supports simultaneous read operations
- Allows write-back during instruction execution
- Register 0 always outputs zero

---

### 4. ALU and ALU Control

The ALU performs arithmetic and logical operations based on control signals generated by the ALU Control unit.

**Supported Operations**
- Addition
- Subtraction
- AND
- OR
- Set Less Than (SLT)

The zero flag generated by the ALU is used for branch decisions.

---

### 5. Data Memory

Handles memory access instructions:
- Load Word (`lw`)
- Store Word (`sw`)

Supports synchronous write and conditional read operations.

---

### 6. Program Counter and Control Flow

The processor supports:

- Sequential execution (PC + 2)
- Branch execution (`beq`)
- Jump instructions (`j`, `jal`)
- Jump Register (`jr`)

Control logic determines the next PC value based on instruction type and ALU flags.

---

### 7. Top Module — Single Cycle MIPS Processor

The `mips_16` module integrates all processor components and performs:

1. Instruction Fetch  
2. Instruction Decode  
3. Execute  
4. Memory Access  
5. Write Back  

This module represents the complete processor datapath and control interaction.

---

### 8. Testbench

The testbench verifies processor functionality by:

- Generating clock signals
- Applying reset conditions
- Running the encoded program
- Observing PC updates and ALU outputs
- Validating instruction execution flow

Simulation confirms correct processor operation.

---

## Instruction Types Executed

The encoded program demonstrates:

- **R-Type Instruction** — Arithmetic operation using registers  
- **Immediate-Type Instruction** — Immediate arithmetic/comparison  
- **Branch-Type Instruction** — Conditional branching and looping
